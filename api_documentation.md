# PayFlow Backend API Documentation

This document outlines all HTTP API endpoints available in the PayFlow backend, organized by module/role.

---

## 1. Authentication APIs

These endpoints are used for logging in and verifying user session status.

| Endpoint | HTTP Method | Request Body / Schema | Description |
| :--- | :--- | :--- | :--- |
| `/login/admin` | `POST` | `LoginRequest`<br>• `email` (string)<br>• `password` (string) | Authenticates administrators and returns a session token. |
| `/login/merchant` | `POST` | `LoginRequest`<br>• `email` (string)<br>• `password` (string) | Authenticates merchants and returns a session token. |
| `/auth/verify` | `GET` | *None* (Requires `Bearer` token header) | Checks the validity of the current session token. |

---

## 2. Public / Shared APIs

These endpoints are publicly accessible without authentication.

| Endpoint | HTTP Method | Request Parameters / Body | Description |
| :--- | :--- | :--- | :--- |
| `/` | `GET` | *None* | Root status check to confirm if the backend is running. |
| `/db-check` | `GET` | *None* | Verifies if the MongoDB connection is alive. |
| `/hello/{name}` | `GET` | Path Param:<br>• `name` (string) | Greeting endpoint for testing connectivity. |
| `/contact` | `POST` | `ContactInquiry`<br>• `name` (string)<br>• `phone` (string)<br>• `email` (string)<br>• `username` (string)<br>• `user_id` (string, optional) | Submits a sign-up application/demo inquiry. |
| `/checkout/{payment_id}` | `GET` | Path Param:<br>• `payment_id` (string) | Retrieves active payment link details for the hosted checkout page. |
| `/checkout/{payment_id}/simulate` | `POST` | Path Param:<br>• `payment_id` (string)<br><br>JSON Body:<br>• `status` (string: `"success"`, `"pending"`, `"failed"`) | Simulates payment execution and triggers websocket status updates. |

---

## 3. Merchant Panel APIs

These endpoints are restricted to authenticated merchants (`Bearer` token header containing a merchant session).

| Endpoint | HTTP Method | Request Body / Schema | Description |
| :--- | :--- | :--- | :--- |
| `/merchant/profile` | `GET` | *None* | Retrieves profile details of the logged-in merchant. |
| `/merchant/profile` | `POST` | `MerchantProfile`<br>• `name` (string)<br>• `email` (string)<br>• `business_name` (string, optional)<br>• `tax_id` (string, optional) | Updates the profile details of the merchant. |
| `/merchant/password` | `POST` | `PasswordUpdate`<br>• `email` (string)<br>• `current_password` (string)<br>• `new_password` (string) | Updates the password of the merchant. |
| `/merchant/regenerate-keys` | `POST` | `RegenerateKeysRequest`<br>• `email` (string) | Invalidates old merchant credentials and returns new merchant API and salt keys. |
| `/merchant/stats` | `GET` | *None* | Retrieves the merchant's metrics (total volume, success rate, links). |
| `/merchant/users/{user_id}` | `GET` | Path Param:<br>• `user_id` (string) | Fetches user details by ID. |
| `/merchant/wallet/history` | `GET` | *None* | Fetches the credit/debit transaction ledger for the merchant's wallet. |
| `/merchant/payments` | `GET` | *None* | Lists all payment links generated by the merchant. |
| `/merchant/payments/{payment_id}` | `GET` | Path Param:<br>• `payment_id` (string) | Fetches the detailed fields of a specific payment link. |
| `/merchant/payments` | `POST` | `Payment`<br>• `name` (string)<br>• `amount` (string)<br>• `currency` (string, default: `"INR"`) <br>• `status` (string, default: `"Active"`) <br>• `upi_id` (string, default: `"nexify@okicici"`) <br>• `custom_qr_link` (string, optional)<br>• `order_id` (string, optional)<br>• `return_url` (string, optional)<br>• `user_id` (string, optional) | Generates a new hosted payment page link (with Cashfree integration). |
| `/merchant/payments/{payment_id}/refund` | `POST` | Path Param:<br>• `payment_id` (string) | Manually marks a payment status as `"Refunded"`. |
| `/merchant/payments/{payment_id}/flag` | `POST` | Path Param:<br>• `payment_id` (string) | Marks a payment status as `"Flagged"` (security alert). |
| `/merchant/verify-utr` | `POST` | JSON Body:<br>• `utr` (string: 12-digit numeric)<br>• `payment_id` (string) | Submits a customer UTR ID for verification against a payment link. |
| `/merchant/withdraw` | `POST` | `WithdrawRequest`<br>• `amount` (float)<br>• `bank_account` (string, optional)<br>• `ifsc_code` (string, optional)<br>• `remarks` (string, optional)<br>• `type` (string: `"bank_transfer"` or `"self_withdrawal"`) | Submits a payout withdrawal request from the merchant's wallet. |
| `/merchant/withdrawals/sync-status` | `POST` | *None* | Queries Cashfree Payout status for pending transfers and updates the database. |
| `/merchant/settlements` | `GET` | *None* | Lists historical settlements for the merchant. |
| `/merchant/customers` | `GET` | *None* | Lists all customers registered under this merchant. |
| `/merchant/customers` | `POST` | `Customer`<br>• `name` (string)<br>• `email` (string)<br>• `merchant_email` (string)<br>• `totalSpend` (string, optional)<br>• `orders` (int, optional)<br>• `joined` (string, optional) | Registers a new customer under this merchant. |
| `/merchant/customers/{customer_id}` | `DELETE` | Path Param:<br>• `customer_id` (string) | Deletes a customer record. |

## 4. Admin Panel APIs

These endpoints are restricted to authenticated system administrators (`Bearer` token header containing an admin session).

| Endpoint | HTTP Method | Request Body / Schema | Description |
| :--- | :--- | :--- | :--- |
| `/admin/merchants` | `GET` | *None* | Lists all registered merchant accounts. |
| `/admin/merchants/{email}` | `DELETE` | Path Param:<br>• `email` (string) | Deletes a merchant account. |
| `/admin/merchants/{merchant_id}/wallet/add` | `POST` | Path Param:<br>• `merchant_id` (string)<br><br>JSON Body (`AddFundsRequest`):<br>• `amount` (float) | Manually adds funds to a merchant's wallet balance. |
| `/admin/inquiries` | `GET` | *None* | Lists all pending sign-up/demo inquiries. |
| `/admin/inquiries/{email}` | `DELETE` | Path Param:<br>• `email` (string) | Rejects and deletes a sign-up inquiry. |
| `/admin/activate-merchant` | `POST` | `ActivateRequest`<br>• `inquiry_id` (string)<br>• `password` (string) | Approves a sign-up inquiry, generating merchant credentials. |
| `/admin/create-merchant` | `POST` | `CreateMerchantRequest`<br>• `name` (string)<br>• `email` (string)<br>• `password` (string)<br>• `username` (string, optional)<br>• `plan` (string, default: `"Standard"`) <br>• `role` (string, default: `"merchant"`) <br>• `wallet_balance` (float, default: `0.0`) | Creates a merchant or admin account directly. |
| `/admin/logs` | `GET` | *None* | Fetches administrative audit logs. |
| `/admin/invites` | `GET` | *None* | Lists all active merchant invitations. |
| `/admin/invites` | `POST` | `MerchantInvite`<br>• `name` (string)<br>• `email` (string)<br>• `amount` (string, optional) | Sends an invitation link / creates a pre-populated payment request. |
| `/admin/invites/{email}` | `DELETE` | Path Param:<br>• `email` (string) | Cancels/deletes a pending invitation. |
| `/admin/create-payment-link` | `POST` | JSON Body:<br>• `username` (string)<br>• `payment` (object containing payment details) | Generates a payment link on behalf of a merchant. |
| `/admin/tickets` | `GET` | *None* | Lists all platform support tickets. |
| `/admin/tickets` | `POST` | `SupportTicket`<br>• `subject` (string)<br>• `merchant` (string)<br>• `priority` (string: `"Low"`, `"Medium"`, `"High"`) <br>• `status` (string, default: `"Open"`) <br>• `message` (string) | Submits a new administrative support ticket. |
| `/admin/tickets/{ticket_id}/reply` | `POST` | Path Param:<br>• `ticket_id` (string)<br><br>JSON Body (`SupportTicketReply`):<br>• `reply_text` (string)<br>• `status` (string, default: `"Resolved"`) | Adds a reply response to a ticket and updates its status. |
| `/admin/stats` | `GET` | *None* | Fetches administrative metrics (total volume, ticket ratios, merchant counts). |
| `/admin/settings` | `GET` | *None* | Retrieves current platform settings. |
| `/admin/settings` | `POST` | `AdminSettings`<br>• `general` (dict)<br>• `security` (dict)<br>• `api` (dict) | Updates platform settings. |
| `/admin/withdraw` | `POST` | `AdminWithdrawRequest`<br>• `merchant_identifier` (string)<br>• `amount` (float)<br>• `bank_account` (string, optional)<br>• `ifsc_code` (string, optional)<br>• `remarks` (string, optional)<br>• `type` (string) | Performs a manual payout/withdrawal transaction for a merchant. |
| `/admin/withdrawals/sync-status` | `POST` | *None* | Synchronizes platform withdrawals with Cashfree. |
| `/admin/payments` | `GET` | *None* | Lists all payment links generated across the entire platform. |
| `/admin/withdrawals` | `GET` | *None* | Lists all payouts/withdrawals requested across the platform. |

---

## 5. Webhook APIs

These endpoints process external event callbacks.

| Endpoint | HTTP Method | Request Body | Description |
| :--- | :--- | :--- | :--- |
| `/webhook/cashfree` | `POST` | Cashfree Webhook Signature / Event Data Payload | Receives notification updates from Cashfree regarding payment events (e.g., success). |
